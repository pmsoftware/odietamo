Setting Up ODBC Source Connections

Before you can import data from an ODBC source, you must:

  1. Determine a configuration,

  2. Install Oracle DQ ODBC support,

  3. Create or validate ODBC system DSN, and

  4. Create a Oracle DQ Loader Connection to access the ODBC Source.

Determine Configuration

In order to import data from ODBC sources, you must install Oracle DQ ODBC support on a Windows Server.

Oracle DQ ODBC support does not have to reside on the same machine as the Oracle DQ server or client.<ORACLE GRAPHIC PLACE HOLDER>

Create or Validate ODBC System DSN

Have your System Administrator verify the existence of an ODBC system DSN for the ODBC source that you will access to import data. Depending on your site, youmay already have an ODBC system DSN established for the ODBC source. If you do not have one, create one.

Validate Login Access to the ODBC Source

To import data directly into Oracle Data Quality from an ODBC source, a Oracle DQ user must have login access to the ODBC source. Ensure that the login credentials are valid.

Setting Up Oracle/IBM DB2 Data Source Connections

Process Overview

Make sure that you complete all of these steps:

  1. Ensure the Oracle DQ Server Application is installed.

  2. Ensure that the appropriate Oracle/IBM DB2 client is installed on the Oracle DQ server.

  3. Ensure that you can connect to Oracle/DB2 from an Oracle/IBM DB2 client that is installed on the Oracle DQ server machine.

For Oracle, try to connect using Net Assist, or similar application.

For IBM DB2, try to connect through Command Center or Control Center, or a similar application.

  1. For Oracle, find out how the database (that Oracle DQ will access for data imports) is named by checking the name specified in the tnsnames.ora file.

  2. For Oracle DQ Server on UNIX, modify the Oracle Data Profiling file .TSDiscovery to reflect your site configuration. See the task below for details.

  3. Ensure that IBM DB2 and Oracle asynchronous I/O processes have not been removed to increase available memory. This action could result in connectivity failure to these RDBMS applications.

  4. Ensure that you have login access to your IBM DB2 or Oracle data sources.

  5. Define an IBM DB2 or Oracle Loader Connection using the Metabase Manager.

  6. If you installed Oracle or DB2 clients after the Oracle DQ Server installation, reboot the Server to set environment variables.

 

To modify the Oracle Data Profiling file on a UNIX Server

The following steps are only necessary for Oracle DQ Servers on UNIX. If an Oracle/DB2 client is installed on the Oracle DQ Server for Windows, the environment information is automatically located through the system environment variables.

When working with Oracle Data Quality, Inetd accesses the Oracle Data Profiling file to gather environment information. By making the following changes to the Oracle Data Profiling file, you are ensuring that the Oracle Data Quality knows how to access Oracle and IBM DB2 databases.

      1. Log on the Oracle DQ Server as UNIX root user.

      2. Stop the Scheduler.

      3. See “To stop the Oracle DQ Scheduler” on page 11-10.

      4. Go to the home directory of the user that installed Oracle Data Quality.

      5. Type the command ls -al and locate the file named .TSDiscovery.

      6. Open the file named .TSDiscovery for editing.

      7. Add an entry to the file based on the following table:

Relational Database

Add...

 Oracle

 

the entry:

export ORACLE_HOME=<path to Oracle client>

IBM DB2

  

an entry to run the db2profile script that is supplied with IBM DB2. Precede the script with a “.” to ensure the environment gets updated.

For example:. /home/db2inst/lib/db2profile

You may be able to copy the required Oracle and IBM DB2 information from the file named .profile found in the home directory of the user that installed Oracle Data Quality.

 

  1. Save and close the file.

  2. Restart the Scheduler.

  3. See “To start the Oracle DQ Scheduler” on page 3-15.

  4. Return to the installation checklist for your server (Windows on page 1-3; UNIX on page 1-3).

Validate Access to the Relational Source

To import data into Oracle Data Quality directly from IBM DB2 or Oracle, a Oracle DQ user must have login access to IBM DB2 or Oracle. Ensure that you have a valid RDBMS user name and password.

Troubleshooting

Sybase

If you receive the following error when trying to load data from a Sybase source: Remote command failed, [DataDirect][ODBC Sybase Wire Protocol driver][SQL Server]Incorrect syntax near x, make sure that the Sybase ASE ODBC Driver configuration settings (Advanced tab) has the option Enable Quoted Identifiers selected.
